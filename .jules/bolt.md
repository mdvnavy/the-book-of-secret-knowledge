## 2026-04-12 - [Optimize GeneratePassword loop]
**Learning:** [In bash scripts, running loops over external commands using process substitution or command substitution like `$(seq 1 "$_count")` invokes unnecessary subshells per count, causing significant overhead. Further, executing external binaries like `openssl` repeatedly in a loop is extremely slow. Using native bash `for (( i=1; i<="$_count"; i++ ))` constructs and caching binary paths or conditions outside loops (or using batch generation natively via `head -n "$_count"`) avoids massive performance bottlenecks without losing readability.]
**Action:** [When processing repetitive counts in bash functions or aliases, use native bash C-style loops `((i=1;i<=count;i++))` instead of `seq`, and always try to batch block operations such as `/dev/urandom` reads by utilizing `head -n` instead of looping read calls.]
